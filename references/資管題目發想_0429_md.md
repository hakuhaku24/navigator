問 題 一 ： 天 氣 應 變
使⽤者痛點：下雨了不知道去哪、⾼溫⾛不動
我們的解法：中央氣象署 API 即時偵測天氣變化

下雨 → ⾃動推薦路線上室內替代景點
⾼溫 → 推薦有冷氣的停留點與補⽔地點
依旅遊族群推不同備案（親⼦ / 銀髮 / 年輕背包客）
理論依據：
Contingency-Coping Framework：預防性準備比事後應變更有效
Leisure Constraints Theory：結構性限制（天氣）的影響因旅遊族群⽽異

overpass api ⽀援特殊飲食需求篩選（素食、清真、過敏

問 題 五 ： 資 訊 是 否 可 信
使⽤者痛點：部落格資訊過時、各來源說法⽭盾、不知道該信
誰
• 我們的解法：五層可信度架構
• 第⼀層：來源分級（官⽅ > 半官⽅ > 部落格）
• 第⼆層：時間衰減（⾃動標記資訊⽇期）
• 第三層：交叉比對（多源⼀致則提升可信度）
• 第四層：群眾回報（旅遊後使⽤者驗證）
• 第五層：透明呈現（✅官⽅確認 / ⚡近期更新 / ⚠建議確認 ）
